# `steelyard/x402`

`steelyard/x402` adds x402 paid HTTP resource support to Steelyard.

```sh
npm install steelyard
```

The package has two public surfaces:

```ts
import { x402Fetch, x402Payments } from "steelyard/x402";
import { x402Paywall, exactUsdc } from "steelyard/x402/server";
```

The umbrella `steelyard` package re-exports the common names:

```ts
import { x402Fetch, x402Payments, x402Paywall, exactUsdc } from "steelyard";
```

## Buyer API

```ts
await wallet.addInstrument(x402Payments({
  signer,
  networks: ["eip155:84532"],
  assets: ["USDC"],
  schemes: ["exact"]
}));

const fetchPaid = x402Fetch(wallet, {
  maxAmount: { amount: "0.10", currency: "USDC" },
  facilitator: "https://x402.org/facilitator"
});

const response = await fetchPaid("https://api.example.com/paid-weather");
console.log(response.x402?.receipt.transaction);
```

`x402Payments(...)` is the recommended constructor. `x402Exact(...)` also
exists in `steelyard/x402` for developers who want a protocol-specific helper
for the x402 `exact` scheme, but first-run code should use
`x402Payments({ schemes: ["exact"] })`.

## Server API

```ts
const paywall = x402Paywall({
  facilitator,
  routes: {
    "GET /paid-weather": exactUsdc({
      amount: "0.001",
      network: "eip155:84532",
      payTo,
      description: "Paid weather API response"
    })
  }
});
```

The first server adapter exposes a Node `RequestListener`-compatible
`paywall.handler`, so it can be mounted with `createServer(paywall.handler)`.

## Important Boundaries

- x402 is for paid HTTP resources, not cart checkout.
- ACP/UCP checkout remains the commerce checkout path.
- x402 routes do not require Steelyard manifests.
- Signer APIs do not accept raw private key strings.
- A facilitator verifies and settles payment payloads; Steelyard does not
  custody funds or operate a default production facilitator.
- Wallet policy runs before signing.

## Offline Example

The x402 weather example runs fully offline with a mock facilitator:

```sh
pnpm --filter steelyard-example-x402-weather test
pnpm --filter steelyard-example-x402-weather build
```
